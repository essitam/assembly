What is this?

This is a prorgram written in p5.js to explore the Collatz Conjecture. It allows you to input a number and then it plots the number's progression through rules of the Collatz Conjecture (see below). Each step is represented by a point on the canvas going from left to right. The hegiht of the point represents the value of the input of the Collatz function at that point. The y-axis is labeled but the x-axis is not

What is the Collatz Conjecture?

The Collatz Conjecture is a conjecture in Mathematics that states for any positive integer n such that n > 1, iterations through the Collatz function will eventually result in an output of 1.

The Collatz function says that for any integer divisible by two, return that integer divided by 2; For all other integers return 1 more than 3 times that integer. In a program, such as Javascript, it would look something like this

function collatz(n) {
 if (n % 2 === 0) {
  return n / 2;
 } else {
  return (3 * n) + 1;
 }
}

Because of how integers work, this function always returns an integer. Since it also accepts integers as input, it is always possible to feed the results of the Collatz function back into itself. This is what is meant by iterations through the Collatz function.

The conjecture can be summed up as follows. For any integer n where n > 1 if you run n through the Collatz function and then take that result and run that through the Collatz function and then take that result and run that through the Collatz function, and so on and so on, eventually you will end up with 1.

This could happen quickly. For example if you start with 2, 2 is divisible by 2, so you return 2 / 2 which is 1. So that only takes one step.

For 3, 3 is not divisible by 2 so you return 3 * 3 + 1 or 10. Now you put 10 through the Collatz function. Since 10 is divisible by 2 you return 10 / 2 or 5. Now 5 goes through the function. Since 5 is not divisble by 2 you get 3 * 5 + 1 or 16. Now 16 goes through the function and since it is divisble by 2 you get 8 which goes through the function and is divisble by 2 giving 4 then 2 then 1. You can see that this takes many more steps than 2 but still ends up at 1

You can do this process for any valid integer and that is what is supposed to be shown by this sketch.
